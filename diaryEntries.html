Week 9:

When sitting down to code this first draft of my final project webpage, the only thing I wanted it to look like was like a modern intake on the ‘classic’ or ‘stereotypical’ code design style (dark background, green letters, monospace font). So, this was my first step. Before I could overcomplicate things in my head, I simply added the CSS that I needed to add the colors and sizes that I wanted. I thought of using a template to design my webpage, but I thought that would make it less of my own. Although it could make it look more professional, I wanted this page to be entirely of my own making, so I decided to not use any templates or follow any tutorials online, and instead just coded this from a blank canva.

I also decided I wanted to have one continuous page, with a vertical flow where the data story continues further down the page, as opposed to one that has multiple different pages. I decided to do this because it would flow better with how I wanted to display my data, and it would also give a ‘smoother’ feel to the data story.

However, I soon realized that my page was feeling cramped and unorganized with all the information just in one view. So, I opened W3 schools to explore what alternatives I had to stick to my one-page look but still make it more organized. This is where I remembered that tables can be handy for organizing information into ‘blocks’. So, I organized my webpage with tables, playing around with the colors to make sure it fits my theme and using the margins to give it a ‘cleaner’ look, making the different tables quite a part from each other. In the beginning, I wanted to keep them so far a part you could barely see that there was another table under it at first glance, but I quickly saw it looked confusing. So, I ended up settling for a middle-ground. This part was a lot of just testing with trail and error.

I also decided to create the graphs for gender, age, nationality and schooling each in a separate html page. This helped me keep things organized. I used chart.js for most of my charts. Since the data I am using is already my own and I am very familiar with it, making the charts was easier than I expected.

When I was coding the chart for the age bar chart, I intially wanted each bar to have a different color, so that is how I coded it. However, when I was done, I saw that the different colors made the chart look very confusing because the colors did not actually stand for anything (it was all just ages). So, I changed everything back to just one color by just copying and pastying the same rgb code repeatedly. For this reason, my code now looks a little bit weird (a lot of repeated rdg codes) unecessarily. Still, I decided to keep it this way just for the sake of doing the same thing in a ‘different way’, and because it was giving me the same result that I wanted anyway.

Furthermore, the hard part that I notice for me is to properly link my CSS, HTML and JavaScript files together. I believe this is an issue with paths and directories. I am solving this by always keeping all html files in the same folder, all css files in the same folder, and all javascript files in the same folder. It is something that required attention as one misplaced file can break the entire code.

Finally, I thoguht that adding some images would help me acheive a more ‘modern’ and creative look for this design. So, I went on Canva.com and got some visuals from there. I changed and perfected their colors myself so it would fit my design how I wanted. Still, I was struggling to center them, despite looking up on W3 schools (the solution I was reading, which was to add class=center, was not working). So, I decided to leave this one to solve in the later weeks. The same applies to the buttons. I knew I wanted to add some CSS to them to make them look nicer, and I still needed to add eventlisteners and find a way to keep the graph in that same page (instead of opening a new tab when the button was clicked) so I decided to leave that to figure out in the later weeks.

Overall, this week I used mainly concepts learned in Week 2, these being the CSS and all the HTML tags. I was setting the groundwork for more elaborate details to come. I also used concepts from Week 7 and Week 8 to actually make the charts I did using chart.js.
